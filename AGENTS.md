# AGENTS.md

## 项目定位
- 本仓库的目标是训练一个Agent挑战PVZ无尽模式。
- 目前包含 PPO 和 DDQN。

## 工作原则
- 非用户明确要求，不直接修改文件。
- 优先局部修复；只有在现有结构无法满足需求时才新增抽象。
- 代码风格优先级：最小改动、保持兼容、最佳实践。
- 训练入口统一从 `train.py` 进入。
- Python 相关命令默认使用 `.venv\Scripts\python.exe`。
- 代码风格保持与现有仓库一致，避免顺手清理无关代码。
- 说明文件和回答中优先使用项目相对路径。

## Sim/Real 对齐约定
- 真实环境会从 spawn list 中移除部分僵尸类型，目前包括气球、矿工、蹦极、巨人、红眼巨人、扶梯和冰车；sim 对齐时不应把这些作为默认目标。
- 出于简化，当前 sim 不考虑磁力菇，也不建模护甲剥离；路障、铁桶等按总血量处理即可。
- 当前实现禁用小推车，sim 对齐时不要假设小推车可用。
- 当前 sim 不考虑泳池关卡、南瓜、铲子和升级植物；除非明确要求，不要引入这些机制。
- 对齐 sim 数值时优先对齐血量、伤害、冷却、速度等已有机制；不要顺手新增控制、溅射等未实现功能。

## 仓库重点
- `train.py`：训练总入口。
- `train_args.py`：命令行参数。
- `training/`：训练生命周期、日志、checkpoint、metrics 和运行准备。
- `envs/`：环境实现。
- `models/ppo/`：PPO 环境、模型构建和训练入口。
- `models/ddqn/`：DDQN 相关实现。
- `callbacks/`：PPO/训练回调。
- `hook/` / `hook_client/`：DLL 注入与 Hook 通信。

## Git 约定
- 原始来源仓库的默认分支是 `master`。
- 主分支为 `main`。
- 提交信息遵守约定式提交，使用英语。
